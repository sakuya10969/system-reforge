# プロジェクト概要

## プロダクト名

System Reforge

## 概要

COBOLなどのレガシーシステムのソースコードを解析し、業務ロジックの抽出・要件定義の生成を支援するツール。

コード変換ツールではない。ソースコードを「読んで理解する」ことに特化し、以下の変換パイプラインで要件を導出する：

```
ソースコード → 構造化中間データ → 業務ルール → 要件定義
```

## 対象

- COBOLを中心としたレガシーシステムのソースコード
- 将来的に他のレガシー言語（PL/I、RPGなど）への拡張を想定

## 目的

| 目的 | 説明 |
|------|------|
| コード解析 | ソースコードの構造・依存関係を解析 |
| 業務ロジック抽出 | 条件分岐・計算ロジックを業務ルールとして抽出 |
| 要件定義支援 | 抽出した業務ルールを要件定義形式に変換 |
| モダナイゼーション支援 | 現行システムの理解を基にした移行計画策定 |

## これは何でないか

- コード変換ツール（COBOL → Java のような変換はしない）
- リファクタリングツール
- コードを書き換えるツール

## 処理フロー概要

1. ユーザーがZIPファイルでソースコードをアップロード
2. S3に原本保存
3. 解析ジョブを作成
4. ワーカーが非同期で解析実行
5. 構造化データ（AST、依存関係、フロー情報）を生成・DB保存
6. LLMで中間データから意味を抽出（ソースコードは直接渡さない）
7. 結果をDB保存
8. フロントエンドで表示・レビュー

## バックエンド構成

バックエンドのアプリケーションコードは `server/app/` 配下に集約する。`server/` 直下は `pyproject.toml`、`.env`、`alembic/` などの実行基盤のみを置く。

```
server/
├── app/
│   ├── main.py
│   ├── api/
│   ├── application/
│   ├── config/
│   ├── domain/
│   └── infrastructure/
└── alembic/
```

起動コマンド:

```bash
cd server
uv run uvicorn app.main:app --reload --port 8000
```
